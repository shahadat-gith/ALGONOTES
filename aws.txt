Step 1: Authenticate with AWS ECR
We need to log your local Docker application into your AWS account.
Copy and paste this exact command into your PowerShell terminal and hit enter:

aws ecr get-login-password --region ap-south-1 | docker login --username AWS --password-stdin 692299587899.dkr.ecr.ap-south-1.amazonaws.com


Step 2: Build the Production Image
AWS's default command assumes you want to build a regular Dockerfile. However, because we built a highly optimized Dockerfile.prod specifically for AWS Lambda, we need to point to it explicitly using the -f flag.

docker build -f Dockerfile.prod -t algonotes-backend-prod .



Step 3: Tag Your Built Image
Now, let's give the local image a proper "label" so your AWS registry recognizes it:

docker tag algonotes-backend-prod:latest 692299587899.dkr.ecr.ap-south-1.amazonaws.com/algonotes-backend-prod:latest



Step 4: Push to AWS ECR 🚀
Finally, upload the container directly to your cloud registry:

docker push 692299587899.dkr.ecr.ap-south-1.amazonaws.com/algonotes-backend-prod:latest
